Setting the App Icon in a React Native iOS Build Without Xcode
For a React Native project, the App Store icon (1024x1024 PNG) is embedded in the iOS build by placing it in the project's asset catalog. You can do this entirely on Windows/Linux using CLI tools—no Xcode GUI needed. The icon gets processed during the build (via pod install and archiving on a Mac/cloud Mac). This mirrors the MAUI approach but uses RN-specific tools like react-native-make or manual asset replacement. Assumes a bare RN workflow (not Expo, as Expo handles icons via config).
Step 1: Prepare Your Icon

Create a 1024x1024 PNG (RGB, 72 DPI, opaque—no transparency/alpha, as Apple rejects it).
Use a free generator like AppIcon.co or MakeAppIcon to auto-generate all iOS sizes from your 1024px source.
Download the ZIP > Extract > You'll get AppIcon.appiconset folder with ~20 icons + Contents.json.


Step 2: Add the Icon to Your RN Project (CLI on Any OS)

Option A: Automated (Recommended—Uses CLI Tool, No Manual Drudgery)
Install the tool: yarn add -D @bam.tech/react-native-make (or npm i -D @bam.tech/react-native-make).
Run: npx react-native set-icon --path /path/to/your/1024x1024.png --background "#FFFFFF" --platform ios (white background; adjust color if needed).
This auto-generates all sizes, replaces the AppIcon.appiconset in ios/YourProject/Images.xcassets/AppIcon.appiconset, and updates Contents.json.

Clean: cd ios && rm -rf Pods Podfile.lock && yarn install (or npm) to reset CocoaPods cache.

Option B: Manual (If You Prefer Generators)
Navigate to ios/YourProject/Images.xcassets/AppIcon.appiconset.
Delete existing icons (PNG files + Contents.json).
Copy the generated icons from the ZIP's AppIcon.appiconset into this folder (overwrite).
Ensure Contents.json matches (it defines sizes like "29x29@3x": "Icon-87@3x.png").

Validate: Open Contents.json in a text editor—fix any paths if mismatched.


Step 3: Verify Locally (Android Test as Proxy)

Build/run for Android: npx react-native run-android.
Check the icon on emulator/device (RN shares icon logic somewhat). For iOS preview without full build: Inspect ios/YourProject/Images.xcassets/AppIcon.appiconset—all slots should be filled.

Step 4: Build the IPA with Icon Embedded (Remote Mac Required)

RN iOS builds need macOS (Apple's rule—no full bypass). Use a cloud Mac ($20/month, e.g., MacinCloud) or GitHub Actions (free tier). The icon embeds automatically during archiving.


Prep Signing: In Apple Developer portal, get certs/profiles. Edit ios/YourProject.xcodeproj/project.pbxproj (text file): Add DEVELOPMENT_TEAM = YourTeamID and PROVISIONING_PROFILE_SPECIFIER = YourProfile under release config.
CLI Build on Remote Mac:
Remote in (VNC/SSH).
cd /path/to/your/rn/project/ios && pod install (installs deps).
Archive: xcodebuild -workspace YourProject.xcworkspace -scheme YourProject -configuration Release -archivePath ./build/YourProject.xcarchive archive (generates .xcarchive with icon).
Export IPA: xcodebuild -exportArchive -archivePath ./build/YourProject.xcarchive -exportPath ./build -exportOptionsPlist exportOptions.plist (create exportOptions.plist with your signing details).

GitHub Actions Alternative (Fully Automated): Add a .github/workflows/ios-build.yml:YAMLname: Build iOS
on: [push]
jobs:
  build:
    runs-on: macos-13
    steps:
    - uses: actions/checkout@v4
    - name: Setup Node
      uses: actions/setup-node@v4
      with: { node-version: '20' }
    - run: npm ci
    - name: Install Pods
      run: cd ios && pod install
    - name: Build IPA
      run: cd ios && xcodebuild -workspace YourProject.xcworkspace -scheme YourProject -configuration Release -archivePath ../build.xcarchive archive
    - name: Export IPA
      run: cd ios && xcodebuild -exportArchive -archivePath ../build.xcarchive -exportPath ../build -exportOptionsPlist exportOptions.plist
    - uses: actions/upload-artifact@v4
      with: { name: app.ipa, path: build/*.ipa }
Add secrets (certs/profiles) in repo settings. Run > Download IPA.


Step 5: Upload to App Store Connect

On remote Mac: Install Transporter (mas install 1454610196 via CLI).
xcrun altool --upload-app --file path/to/your.ipa --username your-apple-id --password app-specific-password.
Icon auto-pulls in Connect (refresh App Information).

Pitfalls & Fixes

Missing 1024x1024? Apple rejects: Ensure it's in AppIcon.appiconset as AppStore1024pt.png.
Build Fails on Icon? Run pod deintegrate && pod install in ios/.
No Mac? Codemagic.io (free tier: 500 mins/month) or Expo EAS (if switching to managed workflow) builds/uploads for you—upload code, they handle icon from assets.
For TeamHub: Football icon? Generate with ball motifs; test on Android for quick visual.

This gets your icon in the build fast. If errors (e.g., signing), share the log. Sources: AboutReact , Stack Overflow , Codemagic Blog .
